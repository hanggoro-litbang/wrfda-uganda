## Run da_wrfvar
```console
cd /scratch/$USER/WDIR/wrfda_temp
ln -sf /home/$USER/WRF/WRFDAV4.7.0/var/run/VARBC.in  ./VARBC.in
ln -sf /scratch/$USER/WDIR/template/wrfinput_d01 ./fg
ln -sf /home/$USER/WRF/WRFDAV4.7.0/var/run/be.dat.cv3 ./be.dat
ln -sf /home/$USER/WRF/WRFDAV4.7.0/run/LANDUSE.TBL ./LANDUSE.TBL
ln -sf /home/$USER/WRF/WRFDAV4.7.0/var/build/da_wrfvar.exe .
cp /scratch/wido/uganda/namelist.input.3dvar.temp ./namelist.input
ln -sf obs_gts_2023-08-23_12:00:00.3DVAR ./ob.ascii
```
create sbatch file
```console
vi da_wrfvar.sbatch
```
The output is wrfvar_output
```console
#!/bin/bash
#SBATCH -J da_wrfvar.exe # Job name
#SBATCH -o da_wrfvar.out # Name of stdout output file
#SBATCH -N 1 # Total number of nodes requested
#SBATCH -n 20 # Total number of mpi tasks #requested
#SBATCH -t 02:30:00 # Run time (hh:mm:ss) - 2.5 hours
#SBATCH -p riset
#SBATCH --exclusive
# Launch MPI-based executable
ml restore intel
time mpirun ./da_wrfvar.exe
```
submit the job
```console
sbatch da_wrfvar.sbatch
```
## Run update BC
```console
ln -sf /home/$USER/WRF/WRFDAV4.7.0/var/build/da_update_bc.exe .
cp /scratch/$USER/uganda/parame.in .
cp /scratch/$USER/WDIR/template/wrfbdy_d01 .
cp /scratch/$USER/uganda/wrfvar_output.temp .
```
```console
da_update_bc.exe
```
At this stage, you should have the files **wrfvar_output.temp** and **wrfbdy_d01** in your WRFDA working directory. They are the WRFDA updated initial and boundary condition files for any subsequent WRF model runs. To use, link a copy of **wrfvar_output.temp** and **wrfbdy_d01** to **wrfinput_d01** and **wrfbdy_d01**, respectively, in your WRF working directory.
