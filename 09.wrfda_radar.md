We dont need to run obsproc for radar case, ob.radar was generated from the previous tutorial
## Run da_wrfvar
```console
mkdir /scratch/$USER/WDIR/uganda_asim_radar
cd /scratch/$USER/WDIR/uganda_asim_radar
ln -sf /home/$USER/WRF/WRFDAV4.7.0/var/run/VARBC.in  ./VARBC.in
ln -sf /scratch/$USER/WDIR/uganda_noasim/wrfinput_d01 ./fg
ln -sf /home/$USER/WRF/WRFDAV4.7.0/var/run/be.dat.cv3 ./be.dat
ln -sf /home/$USER/WRF/WRFDAV4.7.0/run/LANDUSE.TBL ./LANDUSE.TBL
ln -sf /home/$USER/WRF/WRFDAV4.7.0/var/build/da_wrfvar.exe .
cp /scratch/wido/uganda/namelist.input.3dvar.radar.uganda_d02 ./namelist.input
cp /scratch/dmu/training_data/obradar_uganda/20220411_0000_d02_obradar .
```
The radar data and wrfinput is no match, so we need to make an adjustment (for the sake of this tutorial only)
```console
cp 20220411_0000_d02_obradar 20260709_0000_d02_obradar
sed -E -i 's/2022-04-11_([0-9]{2}:[0-9]{2}:[0-9]{2})/2026-07-09_\1/g' 20260709_0000_d02_obradar
cp 20260709_0000_d02_obradar ob.radar
```
create sbatch file
```console
vi da_wrfvar.sbatch
```
The output is wrfvar_output
```console
#!/bin/bash
#SBATCH -J da_wrfvar.exe # Job name
#SBATCH -o da_wrfvar.out # Name of stdout output file
#SBATCH -N 1 # Total number of nodes requested
#SBATCH -n 20 # Total number of mpi tasks #requested
#SBATCH -t 02:30:00 # Run time (hh:mm:ss) - 2.5 hours
#SBATCH -p riset
#SBATCH --exclusive
# Launch MPI-based executable
time mpirun ./da_wrfvar.exe
```
submit the job
```console
sbatch da_wrfvar.sbatch
```

```console
da_update_bc.exe
```
At this stage, you should have the files **wrfvar_output** and **wrfbdy_d01** in your WRFDA working directory. They are the WRFDA updated initial and boundary condition files for any subsequent WRF model runs. To use, link a copy of **wrfvar_output** and **wrfbdy_d01** to **wrfinput_d02** and **wrfbdy_d01**, respectively, in your WRF working directory.
